# Kjobs — 구직 커뮤니티 웹사이트 제작 요청

아래 사양대로 완전히 동작하는 웹사이트를 만들어 주세요. 배포 설정은 하지 말고, 레플릿 안에서 실행하고 미리보기로 확인할 수 있는 상태까지만 완성해 주세요.

## 1. 프로젝트 개요

- **사이트 이름**: Kjobs
- **성격**: 구직·취업 정보를 나누는 커뮤니티 게시판 사이트. 다만 구직 특화 기능은 필요 없고, 일반적인 커뮤니티 게시판 구조로 만들면 됩니다. "Kjobs"라는 이름과 구직 커뮤니티라는 느낌은 로고 텍스트, 환영 문구, 안내 문구 등 브랜딩에만 반영해 주세요.
- **인증 철학**: 이메일 인증, 소셜 로그인, 휴대폰 인증 등은 일절 사용하지 않습니다. **아이디와 비밀번호만으로 가입하고 로그인**하는 단순한 시스템입니다.

## 2. 기술 스택

- 백엔드: **Node.js + Express**
- 데이터베이스: **SQLite** (파일 기반)
- 프론트엔드: **EJS 서버 렌더링 템플릿 + 일반 CSS** (별도 프론트엔드 프레임워크 사용하지 않음)
- 인증: **express-session**(세션 로그인) + **bcrypt**(비밀번호 해시)
- 모든 화면은 모바일에서도 잘 보이도록 반응형으로 만들어 주세요.

## 3. 접근 정책 (가장 중요한 규칙)

**로그인하지 않은 사용자는 어떤 게시글, 댓글, 목록, 검색 결과도 절대 볼 수 없습니다. 예외는 없습니다.**

- 비로그인 상태에서 접근 가능한 페이지는 딱 두 개뿐입니다: **로그인 페이지**, **회원가입 페이지**.
- 비로그인 상태로 다른 어떤 URL에 접근해도 무조건 로그인 페이지로 리다이렉트하세요. (사이트 첫 진입 시 기본 화면 = 로그인 페이지)
- 이 규칙은 서버 미들웨어 수준에서 강제해 주세요. 화면에서 링크만 숨기는 방식은 안 됩니다.
- 차단(정지)된 계정은 올바른 비밀번호를 입력해도 로그인할 수 없으며, "차단된 계정입니다. 관리자에게 문의하세요." 안내를 표시합니다.

## 4. 회원 시스템

### 4.1 회원가입
- 입력 항목은 4개만: **아이디, 비밀번호, 비밀번호 확인, 닉네임**
- 아이디: 영문 소문자·숫자 4~20자, 중복 불가
- 닉네임: 2~10자, 중복 불가 (글·댓글에 표시되는 이름)
- 비밀번호: 8자 이상
- 가입 즉시 이용 가능 (인증·승인 절차 없음), 가입 완료 후 로그인 페이지로 이동

### 4.2 로그인 / 로그아웃
- 아이디 + 비밀번호로 로그인. 성공 시 게시글 목록(메인)으로 이동
- 실패 시 "아이디 또는 비밀번호가 올바르지 않습니다"라고만 표시 (어느 쪽이 틀렸는지 알려주지 않음)
- 같은 아이디로 5회 연속 로그인 실패 시 5분간 로그인 시도 차단
- 세션 유지 24시간, 로그아웃 버튼으로 즉시 세션 종료
- 비밀번호는 반드시 bcrypt로 해시하여 저장 (평문 저장 금지)

### 4.3 등급 체계 (3단계)

| 등급 | 권한 |
|------|------|
| **일반 회원 (user)** | 글/댓글 열람·검색·작성, 본인 글·댓글 수정/삭제 |
| **관리자 (admin)** | 일반 회원 권한 + 모든 글/댓글 삭제 + 이용자 관리 페이지 접근(계정 차단/차단 해제) |
| **마스터 (master)** | 관리자 권한 전부 + **관리자 임명/해제** (마스터는 사이트에 단 1명) |

### 4.4 마스터 계정
- 서버 최초 실행 시 마스터 계정이 자동 생성되도록 시드(seed) 처리해 주세요.
  - 아이디: `master` / 비밀번호: `master1234` / 닉네임: `운영자`
  - 최초 실행 시 콘솔에 "마스터 계정이 생성되었습니다 (master / master1234) — 로그인 후 비밀번호를 변경하세요"를 출력
- 마스터 계정은 차단당하거나 관리자 해제될 수 없습니다.

## 5. 이용자 관리 페이지 (관리자 전용)

관리자(admin)와 마스터(master)만 접근할 수 있는 **이용자 관리 페이지**를 만들어 주세요. 상단 내비게이션에 관리자 이상에게만 "이용자 관리" 메뉴가 보입니다.

- **전체 이용자 목록 표시**: 아이디, 닉네임, 등급, 가입일, 상태(정상/차단), 작성 글 수
- 목록에서 바로 실행할 수 있는 관리 기능:
  - **계정 차단 / 차단 해제** — 관리자와 마스터 모두 가능. 단, 관리자는 다른 관리자나 마스터를 차단할 수 없고, 마스터는 누구든(마스터 본인 제외) 차단 가능
  - **관리자 임명 / 관리자 해제** — **마스터만 가능**. 일반 회원을 관리자로 올리거나 관리자를 일반 회원으로 내림
- 각 버튼은 실행 전 확인 메시지를 띄우고, 실행 후 목록에 즉시 반영해 주세요.
- 일반 회원이 이 페이지 URL로 직접 접근하면 메인으로 리다이렉트하세요 (서버에서 권한 검사).

## 6. 게시판 기능

- 단일 자유게시판 (카테고리 없음)
- 게시글: 제목, 본문(텍스트), 작성자 닉네임, 작성일시, 조회수. 이미지 첨부는 없음
- 목록: 최신순, 페이지당 20개 페이지네이션, 상세 조회 시 조회수 +1
- 작성·수정·삭제: 본인 글만 수정/삭제 가능. 관리자·마스터는 모든 글 삭제 가능(수정은 불가)
- 삭제 시 확인 메시지 표시

## 7. 댓글 기능

- 게시글 상세 하단에 오래된 순으로 표시
- 본인 댓글만 삭제 가능 (댓글 수정 기능은 없음). 관리자·마스터는 모든 댓글 삭제 가능
- 대댓글(답글) 기능은 없음

## 8. 검색 기능

- 게시글 목록 상단에 검색창 배치 (로그인한 회원만 사용 가능 — 3번 접근 정책에 따라 자동으로 충족됨)
- 검색 대상 선택: **제목 / 제목+내용 / 작성자 닉네임**
- SQLite `LIKE` 검색으로 구현, 결과는 최신순 표시, 결과가 없으면 "검색 결과가 없습니다" 표시
- 검색 결과 화면에도 페이지네이션 적용

## 9. 화면 목록 (총 8개)

1. **로그인** (비로그인 시 기본 진입 화면) — 아이디/비밀번호 폼, 회원가입 링크
2. **회원가입** — 4개 입력 항목, 아이디·닉네임 중복 확인
3. **메인 (게시글 목록)** — 검색창, 글 목록(번호·제목·작성자·작성일·조회수), 페이지네이션, 글쓰기 버튼
4. **게시글 상세** — 제목/작성자/작성일/조회수/본문 + 댓글 목록·작성 폼 + 권한에 따른 수정/삭제 버튼
5. **게시글 작성/수정** — 제목, 본문 입력 폼
6. **검색 결과** — 메인과 동일한 목록 형태 + 검색 조건 표시
7. **이용자 관리 (관리자·마스터 전용)** — 5번 사양대로
8. **공통 레이아웃** — 상단에 "Kjobs" 로고(클릭 시 메인), 로그인 상태 표시(닉네임 + 등급 뱃지), 로그아웃 버튼, 관리자 이상에게만 "이용자 관리" 메뉴

디자인은 깔끔하고 단정한 스타일로, 구직 커뮤니티에 어울리는 신뢰감 있는 톤(예: 남색/파랑 계열 포인트 컬러)을 사용해 주세요.

## 10. 데이터베이스 구조

### users
| 필드 | 타입 | 설명 |
|------|------|------|
| id | INTEGER PK | 내부 식별자 |
| username | TEXT UNIQUE | 로그인 아이디 |
| password_hash | TEXT | bcrypt 해시 |
| nickname | TEXT UNIQUE | 닉네임 |
| role | TEXT | `user` / `admin` / `master` |
| is_blocked | INTEGER | 0=정상, 1=차단 |
| created_at | DATETIME | 가입일시 |

### posts
| 필드 | 타입 | 설명 |
|------|------|------|
| id | INTEGER PK | 게시글 번호 |
| user_id | INTEGER FK | 작성자 |
| title | TEXT | 제목 |
| content | TEXT | 본문 |
| view_count | INTEGER | 조회수 |
| created_at / updated_at | DATETIME | 작성/수정 일시 |

### comments
| 필드 | 타입 | 설명 |
|------|------|------|
| id | INTEGER PK | 댓글 번호 |
| post_id | INTEGER FK | 대상 게시글 |
| user_id | INTEGER FK | 작성자 |
| content | TEXT | 내용 |
| created_at | DATETIME | 작성일시 |

## 11. 완성 기준 (체크리스트)

아래가 모두 동작하면 완성입니다.

- [ ] 비로그인 상태에서 로그인/회원가입 외 모든 URL이 로그인 페이지로 리다이렉트된다
- [ ] 아이디/비밀번호/닉네임만으로 가입하고 즉시 로그인할 수 있다
- [ ] 최초 실행 시 마스터 계정(master)이 자동 생성된다
- [ ] 글 작성·수정·삭제, 댓글 작성·삭제가 권한 규칙대로 동작한다
- [ ] 제목/제목+내용/닉네임 검색이 동작한다
- [ ] 마스터로 로그인하면 이용자 관리 페이지에서 관리자 임명·해제, 계정 차단·해제가 된다
- [ ] 관리자로 로그인하면 계정 차단은 되지만 관리자 임명 메뉴는 보이지 않는다
- [ ] 차단된 계정은 로그인이 거부된다
- [ ] 로그인 5회 실패 시 5분간 시도가 차단된다
- [ ] 모바일 화면에서도 레이아웃이 깨지지 않는다

배포 설정(Deployments)은 하지 마세요. 레플릿 실행(Run) 후 미리보기에서 위 체크리스트를 확인할 수 있으면 됩니다.
